# Workflow: Weekly Review

**Trigger:** User says "weekly review" — or runs automatically on [DAY] [TIME]

---

## Pre-flight

Before starting, read:
- `context/business.md` — current goals and priorities
- `memory/leads.md` — current pipeline
- `memory/clients.md` — active client status
- `memory/metrics.md` — recent numbers

---

## Steps

### 1. Check Gmail — Last 7 Days
Using Google Workspace skill, scan inbox for:
- New leads — anyone enquiring about services
- Client comms — updates, questions, approvals needed
- Unanswered emails — anything that needed a reply
- Opportunities — referrals, partnerships, warm intros

Flag by category: Leads | Clients | Ops | Urgent

### 2. Check Calendar — Last 7 Days + Next 7 Days
- What meetings happened last week? Key outcomes?
- What's coming up this week? Any prep needed?
- Any deadlines in the next 7 days?

### 3. Review Pipeline
From `memory/leads.md`, for each lead:
- Where are they in the funnel?
- When was last contact?
- What's the next action and is it overdue?
- Any leads that have gone cold?

### 4. Review Client Status
From `memory/clients.md` + client folders:
- Are all active clients receiving work?
- Any at risk?
- Any reporting due this week?

### 5. Check Metrics
From `memory/metrics.md`:
- Revenue this month vs last month
- Pipeline value
- Any client KPIs needing attention

### 6. Review Outreach & Content Performance

**Cold email (from sending tool / `memory/metrics.md`):**
- Emails sent this week
- Open rate (target: 50%+)
- Reply rate (target: 5%+)
- Positive replies / discovery calls booked
- Action: pause underperformers, launch new variants

**LinkedIn (from platform analytics):**
- Posts published this week
- Top-performing post (impressions, engagement)
- New connections sent vs accepted
- DMs sent vs replies
- Conversations moved to discovery
- Action: double down on what worked, adjust what didn't

**Upwork:**
- Proposals sent this week
- Responses received
- Interviews / contracts started
- Action: refine cover letter if response rate is low

**Networking:**
- Events attended
- Follow-ups sent
- Leads generated

### 7. Produce Summary

```
Weekly Review — [Date]

WINS THIS WEEK
- [What went well]

BLOCKERS
- [What's stuck or needs attention]

PIPELINE STATUS
| Lead | Status | Last Contact | Next Action |
|------|--------|-------------|------------|

CLIENT STATUS
| Client | Health | Outstanding | Due |
|--------|--------|-------------|-----|

OUTREACH STATS
| Channel | Volume | Key metric | Calls booked |
|---------|--------|------------|-------------|
| Cold email | X sent | X% reply rate | X |
| LinkedIn | X posts, X DMs | X replies | X |
| Upwork | X proposals | X responses | X |
| Networking | X events | X follow-ups | X |

CONTENT PERFORMANCE
- Top post: [title] — [impressions] impressions, [engagement]
- Posts published: X/5 target
- Learnings: [what worked, what to change]

TOP 3 PRIORITIES THIS WEEK
1. [Priority]
2. [Priority]
3. [Priority]

FOLLOW-UPS NEEDED
- [Name] — [what and by when]
```

### 7. Update Memory
- Update `memory/leads.md` with any status changes
- Update `memory/metrics.md` with current numbers
- Log in `logs/tasks.md`: Weekly Review — [date] — Completed

---

## Expected Output
A structured weekly summary covering pipeline, clients, wins, blockers, and priorities.

---

## Edge Cases

| Situation | Action |
|-----------|--------|
| Gmail not accessible | Note the gap; produce the rest from local files |
| No leads in pipeline | Flag as priority — outreach or content push needed |
| All metrics are empty | Flag as RED — data collection needs to start |
| Client is overdue for a report | Add to priorities; escalate if more than 2 weeks late |
